# Engineering automation

`eng/` contains deterministic build, test, CI, release, and policy orchestration. Provider-specific CI files call these stable entrypoints instead of reimplementing the quality gate.

This area may validate dependency locks, architecture, contracts, migration checksums, forbidden commercial scope, secret leakage, SBOM/provenance, and release manifests. Negative source scanners use the locked Node runtime and fail closed on missing roots or read failures instead of depending on undeclared runner utilities. Traceability evidence is resolved canonically inside the repository, opened with `O_NOFOLLOW`, classified and read through one stable file descriptor, and covered by path-replacement and cleanup probes. It contains no business logic and is not a runtime dependency.

`policies/validate-traceability.mjs` validates the Release 1 DEC/AC evidence index without redefining contract text. It requires all 42 decisions and 45 acceptance criteria, validates each primary WS against the authoritative matrix, locks the reviewed direct-Epic map, resolves local evidence to an actual xUnit method or exact quality-gate command, requires unique named future tests, and keeps AC-045 slices equal to its top-level Epic coverage. The fast `--structure-only` pass never certifies compiled tests; after `dotnet build`, the quality gate runs `--compiled-tests` and requires every test reference under its expected suite/class in xUnit discovery. External gates stay pending until auditable references exist; URL/ID format checks supplement but do not replace external readback and human approval.

`release/prepare-task-import.mjs` parses all 38 M0–M7 Epic rows from the authoritative execution specification, validates their primary/supporting WS ownership against the system plan and `docs/traceability/delivery-epics.json`, and merges related DEC/AC, contract, evidence, and planned-test references into deterministic JSON/CSV under `artifacts/`. It produces preview-only output while the canonical task-system gate is pending and rejects invented task IDs or owners.

`versions.json` is the machine-readable version review record. Exact SDK and package inputs remain enforced by `global.json`, `Directory.Packages.props`, `.config/dotnet-tools.json`, `.node-version`, the frontend package manifest and lockfiles. Every third-party GitHub Action used by any workflow must be registered there and referenced by its reviewed full commit SHA. CI-installed Docker plugins and security tools are versioned there as well; their installers independently reject malformed locks, build only fixed GitHub HTTPS release URLs, constrain redirect origins and response size, and expose bytes to disk operations only after the reviewed SHA-256 matches. Offline negative probes exercise that fail-closed boundary. M0 infrastructure images use an exact release tag plus a reviewed manifest digest; application image digests are added to the release manifest when immutable images are first built and promoted.

The local quality gate runs frontend coverage plus all six .NET test projects with merged Coverlet JSON. A shared xUnit runner policy makes every skipped test fail, and a conditionally compiled dynamic-skip probe proves that policy instead of relying only on configuration text. The gate requires every production assembly discovered under `src/` to be present, uses the locked Node runtime to fail closed while rejecting C# and frontend coverage-suppression directives in production roots, enforces backend and frontend line coverage, the stricter Domain/Application line and branch thresholds, and count-backed no-decline baselines for quota, authentication, and secret-handling code. A critical path policy may expand but cannot remove Git-base scope. Pull-request CI also evaluates coverable changed source lines across C#, JavaScript, TypeScript, JSX, TSX, and Vue against the pull-request base and fails on missing coverage inventory. A workspace without an initial Git commit or a pull request without production-source changes reports that changed-line check as not evaluated instead of claiming production-line coverage passed.

The initial browser support policy covers the current and previous stable Chromium, Firefox, Safari, and iOS Safari major releases at release qualification, plus the current Firefox ESR. Internet Explorer and legacy embedded WebViews are outside the product boundary. The frontend compiles to ES2022; Playwright runs Chromium smoke coverage for pull requests and Chromium/Firefox/WebKit smoke coverage on `main`.

`.github/workflows/security-evidence.yml` keeps security evidence separate from the build/test gate. It runs CodeQL for C# and JavaScript/TypeScript, production dependency audit, a redacted full-history Gitleaks scan, Trivy source dependency/misconfiguration/license checks, source and candidate-image SBOM generation, candidate-image vulnerability scans, and non-root image metadata verification. The CodeQL job grants only the private-repository upload permissions required by GitHub, including `actions: read`; the version-lock policy rejects permission drift. Evidence uploads are limited to the explicit `artifacts/security` and `artifacts/sbom` directories; generated Compose secret material is never uploaded.

The repository uses `main` as its initial branch and is published to the authorized public `Lyon1984/PoolAI` remote. The bootstrap commit establishes the otherwise-empty remote; subsequent changes must pass through pull requests. GitHub readback confirms that `main` requires the unique `verify`, `browser-smoke`, `dynamic-compose`, `codeql`, `repository-security`, and `image-evidence` checks, applies to administrators, requires resolved conversations, and forbids force pushes and deletion. PR #2 fixed and hardened those paths without suppressions or scanner exclusions, passed all protected checks, and its post-merge main analysis ran 166 rules with zero results: all four alerts became `fixed` automatically, none was dismissed, and no new alert appeared. PR #4 then exercised the production-source path: all six protected checks passed, and `verify` evaluated a non-zero modified-line denominator with exact 1/1 (100%) coverage against the pull-request base.
